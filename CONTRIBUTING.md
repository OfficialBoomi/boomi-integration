# Contributing to boomi-integration

Thank you for your interest in improving the boomi-integration skill!

## How to Contribute

We welcome **issues** — not pull requests.

If you've found a bug, have a feature idea, or see something that could be better, please open a detailed issue. The more context you provide, the faster we can act on it. Great issues include:

- A clear description of the problem or suggestion
- Steps to reproduce (for bugs)
- What you expected vs. what happened
- Relevant error messages, logs, or screenshots
- The AI platform you're using the skill with (Claude Code, Cursor, Codex, etc.)

## Why Not Pull Requests?

We maintain tight standards for this codebase and find it far more efficient to implement changes ourselves using the context from a well-written issue. This keeps the code consistent and avoids lengthy back-and-forth review cycles that slow everyone down.

## Contact

For questions or feedback, reach out to **solutions@boomi.com**.
